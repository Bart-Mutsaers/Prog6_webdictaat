Introductie XAML

XML is een afkorting voor Based Markup language. Even voor de duidelijkheid, je spreek het uit als 'zammel'. Het is een onderdeel van het WPF platform. Je kunt XAML defineren in een .XAML file net als in een .HTML file. Elke XAML file heeft een .CS file waar de 'code-behind' staat.

Je kunt de volgende elementen met XAML defineren:

  • Windows
  • dialog boxes
  • pages
  • user controls!
  • controls
  • Shapes
  • graphics

Je zou de opbouw van een XAML document heel erg kunnen vergelijken met een HTML document. Je bouwt een boom van Elementen en de eigenschappen van deze elementen plaats je in de Attributen. Deze geneste objecten leiden tot 'element tree' die gecompileerd word tot een user interface. 

<Window
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    Title="Window with Button"
    Width="250" Height="100">

    <!-- Add button to window -->
    <Button Name="button">Click Me!</Button>

</Window>

WPF word voornamelijk gebruikt voor deskop applicaties. Daarnaast heb je ook nog Browser hosted applicaties. Deze applicaties hebben de naan XBAP'S. Wij gaan alleen aan de slag met de desktop applicaties.

Je kunt een volledige applicatie opbouwen met verschillende windows en pages. Je kunt zoveel elementen toevoegen als je wil. Je bent daarentegen wel verplicht tot het gebruiken van een Application element.

In een console applicatie heb je een Program class met een Main functie. Dit is de plek waar je applicatie begint. Een dergelijke class heb je ook in XAML. Dit is de Application class en heeft een bijbehorend XAML element wat hem tot stand brengt. In dit element kun je aangeven bij welk window je wil beginnen en eventuele andere logica uitvoeren.

Console App

class Program
{
    static void Main(string[] args)
    {
    }
}

WPF App

<Application
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation" 
    StartupUri="MainWindow.xaml" />    
public partial class App : Application
{

}
